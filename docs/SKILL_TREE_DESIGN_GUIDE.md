# 技能树设计指南

## 概述

本指南帮助您设计高质量的技能树，确保学习路径科学合理、用户体验优秀。

## 设计原则

### 1. 渐进式学习
- 从简单到复杂
- 每个节点建立在前置知识之上
- 避免跳跃式难度提升
- 提供平滑的学习曲线

### 2. 清晰的依赖关系
- 明确前置技能要求
- 避免循环依赖
- 合理的依赖深度（建议不超过 5 层）
- 提供多条学习路径

### 3. 适当的粒度
- 每个节点代表一个独立的学习单元
- 预计学习时间：2-20 小时
- 避免过大或过小的节点
- 可以独立完成和验证

### 4. 平衡的分支
- 提供主线和支线
- 支线提供深度学习机会
- 避免过度分支导致迷失
- 关键路径清晰可见

### 5. 激励机制
- 合理的经验值分配
- 里程碑设置有成就感
- 解锁新内容作为奖励
- 可视化进度反馈

## 节点类型选择

### 基础技能（Basic）
**使用场景**：
- 入门级概念
- 基础语法和工具
- 必备的核心知识
- 后续学习的基石

**特点**：
- 绿色圆形
- 100 EXP
- 1 技能点
- 预计 2-8 小时

**示例**：
- HTML 基础标签
- Python 变量和数据类型
- Git 基本命令
- SQL SELECT 语句

### 进阶技能（Advanced）
**使用场景**：
- 中级概念和技术
- 实用的开发技能
- 需要一定基础的内容
- 专业技能的核心部分

**特点**：
- 蓝色六边形
- 150 EXP
- 2 技能点
- 预计 8-15 小时

**示例**：
- React Hooks
- 数据库索引优化
- 设计模式
- API 设计

### 高级技能（Expert）
**使用场景**：
- 高级主题
- 性能优化
- 架构设计
- 深度专业知识

**特点**：
- 紫色星形
- 200 EXP
- 3 技能点
- 预计 15-20 小时

**示例**：
- 微服务架构
- 机器学习算法
- 编译器原理
- 分布式系统

### 里程碑（Milestone）
**使用场景**：
- 重要的学习阶段完成
- 综合性项目
- 技能认证
- 学习路径的关键节点

**特点**：
- 金色钻石形
- 500 EXP
- 5 技能点
- 预计 20+ 小时

**示例**：
- 完整的 Web 应用项目
- 算法竞赛通过
- 技术认证考试
- 开源项目贡献

## 依赖关系设计

### 依赖类型

#### 1. 强依赖（Dependency）
**定义**：必须完成的前置技能

**使用场景**：
- 概念上的先后关系
- 技术上的必要基础
- 工具链的依赖

**示例**：
```
HTML 基础 → CSS 基础 → 响应式设计
JavaScript 基础 → DOM 操作 → React 入门
```

**视觉表现**：橙色实线箭头

#### 2. 可选依赖（Optional）
**定义**：建议但非必需的前置技能

**使用场景**：
- 相关但不强制的知识
- 提供额外背景的内容
- 替代学习路径

**示例**：
```
Python 基础 ⇢ 数据分析（可选：统计学基础）
Web 开发 ⇢ 性能优化（可选：计算机网络）
```

**视觉表现**：灰色虚线箭头

#### 3. 推荐依赖（Recommended）
**定义**：AI 推荐的学习顺序

**使用场景**：
- 优化的学习路径
- 基于用户数据的推荐
- 动态生成的建议

**视觉表现**：蓝色动画箭头

### 依赖设计最佳实践

1. **避免过度依赖**
   - 每个节点的直接依赖不超过 3 个
   - 总依赖深度不超过 5 层
   - 提供多条到达目标的路径

2. **清晰的依赖链**
   ```
   好的设计：
   A → B → C → D

   避免的设计：
   A → B → C
   ↓   ↓   ↓
   D → E → F
   ↓   ↓   ↓
   G → H → I
   ```

3. **合理的分支点**
   - 在适当的位置提供选择
   - 分支后最终汇聚到里程碑
   - 每个分支都有明确的学习目标

4. **循环检测**
   - 严格避免循环依赖
   - 使用拓扑排序验证
   - 工具自动检测并警告

## 内容设计

### 学习资源配置

#### 资源类型分配
每个节点建议包含：
- 1-2 个主要资源（文档或课程）
- 2-3 个补充资源（文章或视频）
- 1 个参考资源（书籍或文档）

#### 难度匹配
- 基础节点：80% beginner, 20% intermediate
- 进阶节点：20% beginner, 60% intermediate, 20% advanced
- 高级节点：80% advanced, 20% expert

#### 时长估算
- 视频：实际时长 × 1.5（考虑暂停和笔记）
- 文章：字数 ÷ 200（每分钟阅读字数）
- 文档：页数 × 5 分钟
- 课程：官方标注时长 × 1.2

### 练习题设计

#### 数量建议
- 基础节点：3-5 题
- 进阶节点：5-8 题
- 高级节点：8-12 题
- 里程碑：综合项目

#### 难度分布
```
基础节点：
- Easy: 60%
- Medium: 30%
- Hard: 10%

进阶节点：
- Easy: 20%
- Medium: 60%
- Hard: 20%

高级节点：
- Easy: 10%
- Medium: 30%
- Hard: 60%
```

#### 题目类型
1. **选择题**：概念理解
2. **填空题**：语法记忆
3. **编程题**：实际应用
4. **调试题**：问题解决
5. **设计题**：架构能力

### 项目实战

#### 项目规模
- 基础节点：小练习（1-2 小时）
- 进阶节点：中型项目（5-10 小时）
- 高级节点：大型项目（15-20 小时）
- 里程碑：综合项目（30+ 小时）

#### 项目要求
1. **明确的目标**：清晰的功能需求
2. **技术栈**：列出使用的技术
3. **评分标准**：可量化的完成标准
4. **参考资源**：示例代码和文档
5. **扩展挑战**：额外的功能建议

## 布局设计

### 位置规划

#### 垂直布局（推荐）
```
        [起点]
          ↓
      [基础层]
       ↙  ↓  ↘
   [进阶层 A] [进阶层 B]
       ↘  ↓  ↙
      [高级层]
          ↓
      [里程碑]
```

**优点**：
- 清晰的进度感
- 符合从上到下的阅读习惯
- 易于理解学习路径

#### 水平布局
```
[起点] → [基础] → [进阶] → [高级] → [里程碑]
```

**优点**：
- 适合线性学习路径
- 节省垂直空间
- 适合时间线展示

#### 混合布局
```
        [核心路径]
       ↙    ↓    ↘
  [分支A] [分支B] [分支C]
       ↘    ↓    ↙
        [汇聚点]
```

**优点**：
- 灵活性高
- 支持多样化学习
- 适合复杂技能树

### 间距规范

- **节点间距**：
  - 水平：150-200px
  - 垂直：150-200px

- **层级间距**：
  - 主要层级：200-250px
  - 子层级：100-150px

- **分支间距**：
  - 并行分支：200px
  - 可选分支：150px

### 视觉层次

1. **主路径突出**
   - 更粗的连接线
   - 更亮的颜色
   - 动画效果

2. **支线弱化**
   - 较细的连接线
   - 较淡的颜色
   - 虚线表示

3. **里程碑强调**
   - 更大的节点
   - 发光效果
   - 特殊图标

## 经验值设计

### 基础公式

```typescript
baseExp = {
  basic: 100,
  advanced: 150,
  expert: 200,
  milestone: 500
}

// 调整因子
difficultyMultiplier = {
  easy: 0.8,
  medium: 1.0,
  hard: 1.2
}

// 时长因子
timeMultiplier = Math.min(estimatedHours / 10, 2.0)

// 最终经验值
finalExp = baseExp * difficultyMultiplier * timeMultiplier
```

### 平衡建议

1. **总经验值规划**
   - 完整技能树：5000-10000 EXP
   - 每个等级：500-1000 EXP
   - 里程碑占比：30-40%

2. **进度曲线**
   ```
   前 25%：快速进步（高 EXP/时间比）
   中 50%：稳定增长（标准 EXP/时间比）
   后 25%：挑战阶段（低 EXP/时间比，高成就感）
   ```

3. **奖励机制**
   - 首次完成：+20% EXP
   - 完美完成：+30% EXP
   - 连击奖励：+50% EXP
   - 速度奖励：+10-40% EXP

## 技能点设计

### 获取规则

```typescript
skillPoints = {
  basic: 1,
  advanced: 2,
  expert: 3,
  milestone: 5
}

// 额外奖励
bonusPoints = {
  perfectScore: 1,
  speedBonus: 1,
  helpOthers: 1
}
```

### 消耗规则

1. **解锁可选技能**
   - 基础可选：1 点
   - 进阶可选：2 点
   - 高级可选：3 点

2. **特殊功能**
   - 跳过前置（不推荐）：5 点
   - 重置进度：3 点
   - 自定义路径：10 点

3. **平衡建议**
   - 完成主线获得的技能点应足够解锁 50% 的可选内容
   - 鼓励探索但不强制完成所有内容
   - 提供多种获取技能点的方式

## 测试和验证

### 设计检查清单

- [ ] 所有节点都有明确的学习目标
- [ ] 依赖关系清晰且无循环
- [ ] 每个节点都有足够的学习资源
- [ ] 练习题难度合理且数量适当
- [ ] 时间估算准确（误差 ±20%）
- [ ] 经验值分配平衡
- [ ] 技能点获取和消耗平衡
- [ ] 视觉布局清晰易懂
- [ ] 有明确的里程碑
- [ ] 提供多条学习路径

### 用户测试

1. **可理解性测试**
   - 用户能否理解技能树结构
   - 依赖关系是否清晰
   - 学习目标是否明确

2. **可完成性测试**
   - 时间估算是否准确
   - 难度是否合适
   - 资源是否充足

3. **激励性测试**
   - 用户是否有动力继续
   - 奖励是否有吸引力
   - 进度反馈是否及时

### 数据分析

收集和分析：
- 完成率（每个节点）
- 平均完成时间
- 放弃率和放弃点
- 用户反馈和评分
- 学习路径选择

根据数据优化：
- 调整难度
- 补充资源
- 优化依赖关系
- 改进布局
