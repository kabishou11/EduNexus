## 2. 笔记创建流程

```
用户创建笔记
    │
    ▼
保存到 KBStorage
    │
    ├─→ 生成文档 ID
    ├─→ 保存到 IndexedDB (documents)
    └─→ 返回文档对象
    │
    ▼
触发数据同步 (DataSyncManager)
    │
    ├─→ 创建同步任务
    ├─→ 加入队列
    └─→ 调度处理
    │
    ▼
并行执行三个操作
    │
    ├─→ [1] 内容提取 (ContentExtractor)
    │       ├─→ 提取标签 (#tag)
    │       ├─→ 提取关键词 (TF-IDF)
    │       ├─→ 提取双链 ([[link]])
    │       └─→ 提取代码块和公式
    │
    ├─→ [2] 知识图谱同步 (KGSyncService)
    │       ├─→ 创建图谱节点
    │       │   ├─→ 节点 ID: kg_{documentId}
    │       │   ├─→ 节点名称: 文档标题
    │       │   ├─→ 节点类型: concept
    │       │   └─→ 关键词: 提取的关键词
    │       │
    │       ├─→ 建立节点关联
    │       │   ├─→ 基于标签关联
    │       │   ├─→ 基于双链关联
    │       │   └─→ 基于关键词相似度关联
    │       │
    │       └─→ 保存到 IndexedDB (nodes, edges)
    │
    └─→ [3] 学习追踪 (LearningTracker)
            ├─→ 记录学习事件
            │   ├─→ 事件类型: create
            │   ├─→ 文档 ID
            │   ├─→ 时间戳
            │   └─→ 元数据 (标题、标签、字数)
            │
            ├─→ 更新学习会话
            │   ├─→ 添加到当前会话
            │   └─→ 更新活动时间
            │
            └─→ 更新学习统计
                ├─→ 文档创建数 +1
                └─→ 保存到 IndexedDB (stats)
    │
    ▼
同步完成
    │
    └─→ 更新文档的同步时间戳
```

## 3. 笔记编辑流程

```
用户编辑笔记
    │
    ▼
防抖 (2秒)
    │
    ▼
更新 KBStorage
    │
    ├─→ 更新文档内容
    ├─→ 版本号 +1
    └─→ 更新时间戳
    │
    ▼
触发数据同步
    │
    ├─→ 检查版本号变化
    └─→ 创建同步任务
    │
    ▼
并行执行
    │
    ├─→ [1] 重新提取内容特征
    │       └─→ 更新关键词、标签
    │
    ├─→ [2] 更新知识图谱
    │       ├─→ 更新节点信息
    │       ├─→ 重新计算关联
    │       └─→ 删除过期关联
    │
    └─→ [3] 记录编辑事件
            ├─→ 事件类型: edit
            ├─→ 计算编辑时长
            └─→ 更新统计
    │
    ▼
同步完成
```

## 4. 技能完成流程

```
用户完成技能
    │
    ▼
更新技能状态
    │
    ├─→ status: completed
    ├─→ progress: 100
    ├─→ completedAt: 当前时间
    └─→ 计算实际学习时长
    │
    ▼
触发数据同步 (immediate: true)
    │
    ▼
并行执行
    │
    ├─→ [1] 更新知识图谱
    │       ├─→ 查找关联的图谱节点
    │       ├─→ 更新节点状态: mastered
    │       ├─→ 更新掌握度: 1.0
    │       └─→ 保存节点
    │
    └─→ [2] 记录学习事件
            ├─→ 记录完成事件
            │   ├─→ 事件类型: complete
            │   ├─→ 技能 ID
            │   ├─→ 获得经验值
            │   └─→ 学习时长
            │
            ├─→ 更新学习统计
            │   ├─→ 技能完成数 +1
            │   ├─→ 总经验值 +exp
            │   └─→ 总学习时长 +hours
            │
            └─→ 检查成就解锁
                └─→ 触发成就系统
    │
    ▼
同步完成
    │
    └─→ 显示完成动画和奖励
```

## 5. 知识图谱关联流程

```
文档同步到知识图谱
    │
    ▼
提取文档特征
    │
    ├─→ 标签: ['react', 'hooks', 'useState']
    ├─→ 关键词: ['component', 'state', 'render']
    └─→ 双链: ['React Basics', 'State Management']
    │
    ▼
创建/更新图谱节点
    │
    ├─→ 节点 ID: kg_doc_123
    ├─→ 节点名称: "React Hooks 详解"
    ├─→ 节点类型: concept
    ├─→ 关键词: ['react', 'hooks', 'component', 'state']
    └─→ 文档关联: ['doc_123']
    │
    ▼
建立节点关联 (并行)
    │
    ├─→ [1] 基于标签关联
    │       ├─→ 查找包含 'react' 的节点
    │       ├─→ 查找包含 'hooks' 的节点
    │       └─→ 创建 related 类型的边 (strength: 0.6)
    │
    ├─→ [2] 基于双链关联
    │       ├─→ 查找名称匹配 'React Basics' 的节点
    │       ├─→ 查找名称匹配 'State Management' 的节点
    │       └─→ 创建 related 类型的边 (strength: 0.8)
    │
    └─→ [3] 基于关键词相似度关联
            ├─→ 计算与其他节点的相似度
            ├─→ 选择相似度最高的 3 个节点
            └─→ 创建 related 类型的边 (strength: 0.4)
    │
    ▼
保存到 IndexedDB
    │
    ├─→ 保存节点到 nodes store
    └─→ 保存边到 edges store
    │
    ▼
关联完成
```

## 6. 学习会话管理流程

```
用户开始学习
    │
    ▼
LearningTracker 初始化
    │
    ├─→ 检查是否有未结束的会话
    │   ├─→ 有 → 恢复会话
    │   └─→ 无 → 创建新会话
    │
    └─→ 记录会话开始时间
    │
    ▼
用户进行学习活动
    │
    ├─→ 创建笔记 → 记录事件 → 添加到会话
    ├─→ 编辑笔记 → 记录事件 → 添加到会话
    ├─→ 查看笔记 → 记录事件 → 添加到会话
    └─→ 完成练习 → 记录事件 → 添加到会话
    │
    ▼
检测会话状态 (每次活动)
    │
    ├─→ 更新最后活动时间
    └─→ 检查是否超时 (5分钟无活动)
        ├─→ 是 → 结束当前会话
        │       ├─→ 记录结束时间
        │       ├─→ 计算总时长
        │       ├─→ 计算专注度评分
        │       └─→ 保存会话
        │
        └─→ 否 → 继续当前会话
    │
    ▼
用户关闭页面
    │
    └─→ beforeunload 事件
        ├─→ 结束当前会话
        └─→ 保存未完成的同步任务
```

## 7. 专注度评分算法

```
计算专注度评分
    │
    ├─→ 输入
    │   ├─→ 会话时长 (duration)
    │   └─→ 活动次数 (activityCount)
    │
    ├─→ 计算活动频率
    │   └─→ activityRate = (activityCount / duration) * 60
    │       (每分钟活动次数)
    │
    └─→ 评分规则
        ├─→ 理想频率: 2-5 次/分钟 → 评分 1.0
        ├─→ 低于理想: < 2 次/分钟 → 评分 activityRate / 2
        └─→ 高于理想: > 5 次/分钟 → 评分 5 / activityRate
        (最低评分 0.3)

示例:
- 30分钟，90次活动 → 3次/分钟 → 评分 1.0 (专注)
- 30分钟，30次活动 → 1次/分钟 → 评分 0.5 (一般)
- 30分钟，300次活动 → 10次/分钟 → 评分 0.5 (分散)
```

## 8. 同步任务队列处理

```
同步任务队列
    │
    ├─→ 任务入队
    │   ├─→ 创建任务对象
    │   │   ├─→ id: task_timestamp_random
    │   │   ├─→ type: document | skill | graph
    │   │   ├─→ status: pending
    │   │   ├─→ data: 任务数据
    │   │   ├─→ retryCount: 0
    │   │   └─→ maxRetries: 3
    │   │
    │   └─→ 添加到队列
    │
    ├─→ 调度处理
    │   ├─→ 检查是否正在处理
    │   ├─→ 延迟 100ms 开始处理
    │   └─→ 避免并发冲突
    │
    ├─→ 批量处理
    │   ├─→ 获取待处理任务 (最多 5 个)
    │   ├─→ 并行处理任务
    │   │   ├─→ 任务 1 → 处理 → 完成/失败
    │   │   ├─→ 任务 2 → 处理 → 完成/失败
    │   │   ├─→ 任务 3 → 处理 → 完成/失败
    │   │   ├─→ 任务 4 → 处理 → 完成/失败
    │   │   └─→ 任务 5 → 处理 → 完成/失败
    │   │
    │   └─→ 检查是否还有待处理任务
    │       ├─→ 有 → 继续处理下一批
    │       └─→ 无 → 处理完成
    │
    └─→ 错误处理
        ├─→ 任务失败
        │   ├─→ retryCount < maxRetries
        │   │   ├─→ 是 → 重新入队
        │   │   │   └─→ 延迟重试 (2秒 * retryCount)
        │   │   └─→ 否 → 标记为永久失败
        │   │
        │   └─→ 记录错误日志
        │
        └─→ 任务成功
            ├─→ 从队列移除
            └─→ 记录成功日志
```

## 9. 离线同步流程

```
用户离线编辑
    │
    ├─→ 检测到离线 (navigator.onLine = false)
    │
    ├─→ 正常保存到本地 IndexedDB
    │
    └─→ 同步任务入队 (但不处理)
    │
    ▼
用户继续编辑
    │
    ├─→ 所有操作正常进行
    └─→ 同步任务持续入队
    │
    ▼
页面关闭前
    │
    └─→ beforeunload 事件
        └─→ 保存未完成任务到 localStorage
            └─→ key: edunexus_pending_sync_tasks
    │
    ▼
用户恢复在线
    │
    ├─→ 检测到在线 (online 事件)
    │
    ├─→ 从 localStorage 恢复任务
    │   ├─→ 解析任务列表
    │   ├─→ 重置任务状态为 pending
    │   └─→ 加入同步队列
    │
    └─→ 自动开始同步
        ├─→ 批量处理任务
        ├─→ 显示同步进度
        └─→ 完成后清理 localStorage
    │
    ▼
同步完成
    │
    └─→ 显示同步成功提示
```

## 10. 数据一致性保证

```
数据一致性机制
    │
    ├─→ [1] 版本控制
    │       ├─→ 每次更新版本号 +1
    │       ├─→ 同步时检查版本号
    │       └─→ 避免覆盖新数据
    │
    ├─→ [2] 事务处理
    │       ├─→ IndexedDB 事务
    │       ├─→ 多个操作原子性
    │       └─→ 失败自动回滚
    │
    ├─→ [3] 同步锁
    │       ├─→ 同一文档不并发同步
    │       ├─→ 使用任务队列串行化
    │       └─→ 避免数据竞争
    │
    ├─→ [4] 错误恢复
    │       ├─→ 自动重试机制
    │       ├─→ 失败任务持久化
    │       └─→ 手动重试选项
    │
    └─→ [5] 数据校验
            ├─→ 保存前验证数据
            ├─→ 同步前检查完整性
            └─→ 异常数据隔离
```

## 11. 性能优化策略

```
性能优化
    │
    ├─→ [1] 批量处理
    │       ├─→ 每批处理 5 个任务
    │       ├─→ 减少数据库操作次数
    │       └─→ 提高吞吐量
    │
    ├─→ [2] 防抖机制
    │       ├─→ 编辑防抖 2 秒
    │       ├─→ 减少同步频率
    │       └─→ 降低系统负载
    │
    ├─→ [3] 增量同步
    │       ├─→ 只同步变更数据
    │       ├─→ 检查版本号
    │       └─→ 减少数据传输
    │
    ├─→ [4] 索引优化
    │       ├─→ 多字段索引
    │       ├─→ 复合索引
    │       └─→ 加速查询
    │
    ├─→ [5] 异步处理
    │       ├─→ 非阻塞同步
    │       ├─→ 后台处理
    │       └─→ UI 响应流畅
    │
    └─→ [6] 缓存策略
            ├─→ 内存缓存常用数据
            ├─→ 减少数据库访问
            └─→ 提高响应速度
```
