# PWA 部署检查清单

## 开发环境准备

### 依赖安装
- [ ] 运行 `pnpm install` 安装所有依赖
- [ ] 确认 `web-push` 和 `idb` 已安装
- [ ] 确认 `@types/web-push` 已安装

### VAPID Keys 配置
- [ ] 运行 `node scripts/generate-vapid-keys.mjs` 生成密钥
- [ ] 创建 `.env.local` 文件
- [ ] 添加 `NEXT_PUBLIC_VAPID_PUBLIC_KEY`
- [ ] 添加 `VAPID_PRIVATE_KEY`
- [ ] 确认密钥不要提交到 Git

### 应用图标
- [ ] 准备 logo 源文件（建议 512x512 或更大）
- [ ] 生成 8 个尺寸的图标（72, 96, 128, 144, 152, 192, 384, 512）
- [ ] 将图标放置到 `public/icons/` 目录
- [ ] 验证所有图标文件存在
- [ ] 可选：使用 `node scripts/generate-placeholder-icons.mjs` 生成占位图标

### 本地测试
- [ ] 运行 `pnpm dev` 启动开发服务器
- [ ] 访问 http://localhost:3000
- [ ] 打开 Chrome DevTools > Application
- [ ] 验证 Manifest 配置正确
- [ ] 验证 Service Worker 已注册
- [ ] 检查 Cache Storage
- [ ] 测试离线功能（断网后刷新页面）
- [ ] 访问 `/pwa-test` 测试所有功能
